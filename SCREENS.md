## 3. 画面遷移とアクセス制御（Guard）

### 3.1 未認証ユーザーの制御
- `/login` 以外にアクセスした場合、強制的に `/login` へリダイレクトする。

### 3.2 初回ログインユーザーの制御
- ログイン成功時、APIから `first_login_required = true` が返却された場合、強制的に `/change-password` へリダイレクトする。
- パスワード変更が完了するまで、サイドバーのリンクをクリックしても他の画面（`/` や `/daily-tasks` など）へは遷移させない（ガード処理）。

### 3.3 権限制御
- 役割が `member`（一般部員）のユーザーが `/users` などの管理者専用ルートにアクセスした場合、ダッシュボード `/` へリダイレクトし、エラー通知（トースト等）を表示する。

## 4. 各画面のUI要件・コンポーネント

### SCR-001: ログイン画面
- 中央にログインフォームを配置。
- **入力項目**: メールアドレス、パスワード。
- **アクション**: ログインボタン。エラー時はフォーム上部に赤文字でメッセージ表示。

### SCR-002: 初回パスワード変更画面
- ヘッダー・サイドバーは表示しない（または操作不可にする）。
- **入力項目**: 現在のパスワード（`elpa1234`）、新しいパスワード、新しいパスワード（確認用）。
- **アクション**: 変更してダッシュボードへ進むボタン。

### SCR-003: ダッシュボード
- 画面上部にタブを用意し、「月次目標サマリー」と「日次ルーチンサマリー」を切り替え可能にする。
- **月次タブ**:
  - 全体の年間目標達成率のプログレスバー。
  - 今月の進捗未入力者アラート。
- **日次タブ**:
  - 「未着手」「対応中」「完了」「保留」の件数パネル。
  - 3日以上進捗更新がない「停滞案件」の警告リスト。

### SCR-006: 月次進捗入力画面
- **表示項目**: 対象の年間目標、個人割当の内容、対象年月。
- **入力項目**:
  - 進捗率（0〜100%の数値入力またはスライダー）
  - 今月の進捗コメント（テキストエリア）
  - 遅延理由（テキストエリア・任意）
  - 次月の予定（テキストエリア）
- **アクション**: 保存ボタン。

### SCR-007: 日次業務一覧画面
- **検索・フィルタエリア**:
  - 担当者（デフォルトは自分）、ステータス（複数選択可）、業務種別で絞り込み可能。
- **一覧表示（テーブル形式 または カンバン形式）**:
  - カラム: ステータス、進捗率、件名、業務種別、担当者、最終更新日。
  - リスト行をクリックすると編集画面（SCR-008）へ遷移、またはモーダルで開く。

### SCR-008: 日次業務登録・編集画面
- 新規登録と既存編集で共通のフォームコンポーネントを使用する。
- **入力項目**:
  - 件名、内容、業務種別、依頼元、担当者（ドロップダウン）。
  - **ステータス（ラジオボタンまたはセレクト）**: 未着手 / 対応中 / 完了 / 保留
  - **進捗率（数値入力）**:
    - *UI連動ロジック*: ステータスが「未着手」なら `0` 固定（操作不可）。「完了」なら `100` 固定（操作不可）。「対応中」のときだけ `1〜99` を入力可能にする。
  - 保留理由（テキストエリア）: ステータスが「保留」の時のみ入力枠を表示（必須化）。
- **履歴表示**:
  - 編集画面の下部には、過去のステータス変更・進捗更新の履歴（タイムライン）を降順で表示する。

### SCR-009: ユーザー管理画面（管理者専用）
- ユーザーの一覧テーブル。
- 行の右端に「編集」「無効化」「PW初期化」のアクションボタンを配置。
- 新規追加ボタン押下で、ユーザー登録モーダルを開く。
- PW初期化時は確認ダイアログ（「本当に初期状態(elpa1234)に戻しますか？」）を出す。

## 5. AIへの実装指示（フロントエンド用）
- ルーティングには `react-router-dom` などを利用し、ルートガード（PrivateRoute / GuestRoute等のコンポーネント）を実装すること。
- UIライブラリ（MUI, Chakra UI, Tailwind CSS等）の指定があればそれに従うこと。特に指定がなければ、シンプルで保守しやすい Tailwind CSS または生のCSS Modules を用いること。
- `SCR-008` の日次業務フォームでは、Reactの `useEffect` または `react-hook-form` などの状態管理を用いて、ステータス選択に応じた進捗率の自動計算・入力制限UIを必ず実装すること。

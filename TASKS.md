# TASKS.md
# 情報システム部 テレワーク向け業務進捗管理システム 実装タスク

## 0. 前提
このシステムは以下の要件を満たすこと。

- フロントエンドは React で構築する
- Webブラウザから利用する社内システムとする
- 利用者はメールアドレスをログインIDとして使用する
- 初期パスワードは `elpa1234` とする
- 初回ログイン時に必ずパスワード変更を要求する
- 部門目標は年間目標であり、月次で進捗確認を行う
- 日々のルーチン業務は、未着手・対応中・完了・保留で管理する
- 日々のルーチン業務で「対応中」の場合は進捗率を入力できる
- 権限は少なくとも「一般部員」「管理者」を持つ
- 認証と権限制御はフロントエンドだけでなくバックエンドでも実施する

## 12. AIへの追加指示
以下の方針で作業してください。

- 1回で全部実装しようとせず、機能ごとに分割して進めること
- まずは最小構成で動く画面とAPIを作ること
- 各ステップで必要なファイルだけ変更すること
- 大きな変更前には、変更対象ファイル一覧と実装方針を提示すること
- DBスキーマ変更時は migration または SQL 定義を必ず提示すること
- フロントエンドとバックエンドの責務を混ぜないこと
- セキュリティに関わる処理はサーバー側でも必ず検証すること
- 初期パスワードは登録時のみ使用し、初回変更必須の実装にすること
- 日次ルーチン業務の進捗率はステータスに応じて自動制御すること
